Core RL Algorithms Summary (core_algos.py)

This file is a comprehensive collection of Reinforcement Learning (RL) core algorithms and utilities, primarily focused on Policy Gradient methods used in large language model (LLM) alignment (like PPO, GRPO, RLOO, etc.).

1. Overview

core_algos.py provides implementations for:

  • KL Divergence Controllers: Managing the balance between the new policy and the reference policy.
  • Advantage Estimators: Multiple ways to calculate how much “better” an action is compared to a baseline.
  • Policy Loss Functions: Various objectives for updating the policy (Actor).
  • Value Loss Functions: Objectives for updating the value function (Critic).
  • Reward Utilities: Calculating token-level rewards with KL penalties.

2. Key Components

A. KL Controllers

Used to dynamically or statically adjust the KL penalty coefficient during training.

  • AdaptiveKLController: Implements the adaptive KL control from the “Learning to Summarize with Human Feedback” paper. It adjusts the KL coefficient based on a target KL value.
  • FixedKLController: Uses a constant KL coefficient.

B. Advantage Estimators

The file contains several ways to estimate the “Advantage” ($A_t$), which is crucial for reducing variance in policy gradients.

  • compute_gae_advantage_return: The standard Generalized Advantage Estimation (GAE). Uses a value network to calculate temporal difference errors.
  • compute_grpo_outcome_advantage: Implementation of Group Relative Policy Optimization (GRPO). Calculates advantages by comparing rewards within a group of samples for the same prompt, often used for DeepSeek-style training.
  • compute_rloo_outcome_advantage: Reinforce Leave-One-Out (RLOO).
  • compute_remax_outcome_advantage: ReMax estimator.
  • compute_opo_outcome_advantage: Online Preference Optimization (OPO).
  • compute_gpg_outcome_advantage: Group Policy Gradient (GPG).

C. Policy Loss Functions (Actor)

These functions compute the loss used to update the policy network.

  • compute_policy_loss: The standard PPO Clipped Objective. Prevents the policy from changing too drastically in one step.
  • compute_policy_loss_reinforce: Standard REINFORCE (Policy Gradient) loss.
  • compute_policy_loss_gspo: GSPO loss.
  • compute_policy_loss_clip_cov & compute_policy_loss_kl_cov: Experimental/alternate RL loss formulations.

D. Value Loss Function (Critic)

  • compute_value_loss: Computes the Mean Squared Error (MSE) between predicted values and actual returns, typically clipped to ensure stability, as in the standard OpenAI PPO implementation.

3. Core Logic Flow

  1. Reward Calculation: compute_rewards adds the KL penalty to the scores provided by the reward model.
  2. Advantage Calculation: Depending on the algorithm (e.g., GAE or GRPO), advantages are calculated for each token or sequence.
  3. Loss Computation:
    • Actor Loss: compute_policy_loss uses the advantages and log-probabilities to find the direction to update the network.
    • Critic Loss: compute_value_loss updates the value network to better predict future rewards.

4. Design Patterns

  • Registry System: the file uses decorators (@register_policy_loss, @register_adv_est) to register different loss functions and advantage estimators. This allows for easy switching of algorithms via configuration files.
  • Vectorization: Many functions (like compute_grpo_vectorized_outcome_advantage) are optimized for GPU execution using PyTorch tensor operations.

5. Quick Reference for Reading Code

  • Top of file: Registries and KL Controllers.
  • Middle (Lines 200-750): Large collection of Advantage Estimators.
  • Middle (Lines 840-1400): Policy Loss Implementations ($L_{CLIP}$, $L_{PG}$).
  • Bottom: Value Loss, KL penalty math, and data reweighting logic.
